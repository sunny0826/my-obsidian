# Issue: 2 个 Cron 任务超时问题

## 问题描述

2 个 Cron 任务因超时导致 error 状态：

| 任务 | 状态 | 超时时间 | 原因 |
|------|------|----------|------|
| 趋势研究-晚间 | error | 250s > 120s | 任务执行时间过长 |
| 整理记忆 | error | 119s ≈ 120s | 接近默认超时限制 |

## 根因分析

1. **趋势研究-晚间**: 任务需要抓取多个平台（X、Reddit、GitHub），执行时间较长（约 4-5 分钟）
2. **整理记忆**: 需要读取多个 memory 文件并合并，接近超时限制

## 建议解决方案

1. **增加超时时间**: 
   - 趋势研究-晚间：增加到 600s (10 分钟)
   - 整理记忆：增加到 300s (5 分钟)

2. **优化任务逻辑**:
   - 趋势研究：考虑分批抓取或缓存结果
   - 整理记忆：减少读取文件数量或优化合并逻辑
